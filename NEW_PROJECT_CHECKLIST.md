# 새 프로젝트 시작 체크리스트

이 문서는 이 저장소의 스킬 세트를 다른 작품용 새 프로젝트로 옮겨 쓸 때 확인할 항목을 정리한 문서다.

## 1. 프로젝트 생성

- 새 작품용 폴더 또는 새 Git 저장소를 만든다.
- 독립 작품이면 기존 작품과 저장소를 분리한다.
- 같은 시리즈가 아니면 기존 작품 초안과 설정 파일을 가져오지 않는다.

## 2. 공용 자산 복사

새 프로젝트에 아래 항목을 복사한다.

- `skills/`
- `Platform/`
- 필요하면 루트 `README.md`

주의:
- `drafts/`는 기존 작품 파일이 없는 상태로 새로 시작하는 편이 안전하다.
- 기존 `AGENTS.md`는 플랫폼 선택 후 적용본으로 교체하는 것이 좋다.

## 3. 플랫폼 선택

먼저 사용할 런타임 하나를 고른다.

- OpenAI Codex
- OpenAI GPT-OSS
- Google Antigravity
- Anthropic Claude

주의:
- 여러 플랫폼용 운영 파일을 동시에 루트에 섞어 두지 않는다.
- 한 프로젝트에는 한 시점 기준으로 하나의 플랫폼 적용본만 유지한다.

## 4. 플랫폼 파일 적용

선택한 플랫폼의 README를 먼저 읽고 루트에 필요한 파일을 적용한다.

- Codex: `Platform/OpenAI/Codex/README.md`
- GPT-OSS: `Platform/OpenAI/GPT-OSS/README.md`
- Antigravity: `Platform/Google/Antigravity/README.md`
- Claude: `Platform/Anthropic/Claude/README.md`

적용 후 확인:

- 루트 `AGENTS.md` 또는 해당 플랫폼의 시스템 프롬프트 파일이 준비됐는지 확인
- `skills/*/SKILL.md`를 현재 플랫폼이 읽는 방식에 맞게 유지했는지 확인
- `skills/*/references/`가 함께 포함됐는지 확인

## 5. 작품 전용 자료 초기화

새 프로젝트에서는 작품 전용 문서를 새로 만든다.

권장 문서:

- `project/characters.md`
- `project/world.md`
- `project/plot.md`
- `project/rules.md`

최소 기준:

- 인물
- 세계관
- 줄거리 또는 시즌 구조
- 금지 클리셰 또는 문체 금지사항

## 6. 초안 폴더 준비

초고 저장 경로를 새 작품 기준으로 비워 두거나 새로 만든다.

권장 구조:

- `drafts/초고`
- `drafts/개고`
- `drafts/원고`

주의:
- 이전 작품 파일이 남아 있으면 검색과 문맥 회수에서 섞일 수 있다.
- 파일명 규칙은 플랫폼 문서 또는 루트 작업 규칙에 맞춘다.

## 7. 첫 실행 전 점검

- 현재 프로젝트가 어떤 작품용인지 루트 `README.md`에 적는다.
- 적용한 플랫폼 이름을 적는다.
- 테스트 요청 하나로 라우팅이 맞는지 확인한다.

권장 테스트:

- 대사 수정 요청이 `character-voice-bible` 성격으로 처리되는지 확인
- 서술/장면 본문 요청이 `novel-writing` 성격으로 처리되는지 확인
- 장기 설계 요청이 `longform-story-design`으로 분기되는지 확인
- 점검/진단 요청이 `series-qa`로 분기되는지 확인

## 8. 운영 중 주의사항

- 다른 작품을 시작하면 같은 저장소에 섞지 말고 다시 분리한다.
- 플랫폼을 바꿀 때는 해당 플랫폼 README의 전환 가이드를 먼저 따른다.
- 공용 스킬 문서를 수정했다면, 그 수정이 작품 전용 조정인지 공용 개선인지 구분해서 커밋한다.

## 빠른 요약

새 프로젝트에서는 공용 스킬 세트만 복사하고, 작품 자료와 초안은 새로 시작하면 된다. 그리고 플랫폼은 하나만 골라 적용하는 것이 가장 안전하다.
