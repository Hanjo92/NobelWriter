# Local Platform Pack

이 폴더는 Ollama를 포함한 로컬 LLM 호스트에서 NobelWriter를 사용할 때 필요한 운영 문서를 정리하는 공간입니다.

## 목적

- 로컬 호스트별로 다른 제약을 문서화
- 파일 저장, 도구 호출, 메타데이터 주입이 불안정한 환경에서도 집필이 멈추지 않게 보완
- 공용 `skills/` 자산은 유지하고, 호스트별 주입 문구만 따로 관리

## 하위 구조

- [Ollama](./Ollama)
  - Ollama 기반 로컬 실행 환경용 문서

## 공통 원칙

- 로컬 호스트는 `agents/openai.yaml` 같은 메타데이터를 그대로 쓰지 못할 수 있다.
- 파일 저장 도구가 없더라도 한국어 소설 본문 작성은 계속되어야 한다.
- 스킬 호출 문법이 없어도, 요청 의미를 보고 적절한 작업층을 내부적으로 선택해야 한다.

## 권장 흐름

1. 현재 로컬 호스트가 어떤 프롬프트 주입 방식을 쓰는지 확인한다.
2. 해당 하위 폴더의 `SYSTEM_PROMPT` 또는 fallback 문서를 호스트 설정에 반영한다.
3. 공용 자산인 루트 `skills/`, `skills/*/references/`, `drafts/` 구조는 가능한 유지한다.

전용 CLI 유무나 자연어 요청 방식이 헷갈리면 [USAGE_FAQ.md](../../USAGE_FAQ.md)를 먼저 확인하세요.
