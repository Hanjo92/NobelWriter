# Voice Design Workflow

Use this file when the user needs the full order for designing or repairing a cast voice system.

## Workflow Order

1. Identify the dialogue failure.
2. Choose the characters that actually need profiles.
3. Lock each speaker's baseline register and status handling.
4. Define pressure behavior and emotional leakage.
5. Map relationship-dependent variants.
6. Compare overlapping speakers.
7. Audit sample dialogue and repair drift.
8. Deliver reusable rules for future drafting.

## Start From The Smallest Useful Cast

Do not profile the whole ensemble by default. Prioritize:

- protagonist
- antagonist or main counterforce
- romantic counterpart if relevant
- recurring support cast who carry dialogue load

Minor cast can inherit a short rule set until they matter.

## Build Voice From Scenes, Not Adjectives

Avoid labels such as "cool," "cute," or "blunt" without behavioral proof. Define:

- how they refuse
- how they ask for help
- how they hide fear
- how they test another person
- how they react when status is challenged

Scene behavior will hold up longer than descriptive tags.

## Test In Dialogue Pairs

Before finalizing a voice card, test the speaker against at least two different counterparts:

- someone above them
- someone emotionally dangerous to them

If the voice only works in one pairing, it is not stable enough.

## Repair Order For Existing Manuscripts

When revising accumulated chapters:

1. fix honorific and address mismatches
2. remove authorial exposition from dialogue
3. separate overlapping reaction styles
4. restore pressure-based line movement
5. update voice rules for future chapters
