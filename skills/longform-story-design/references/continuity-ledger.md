# Continuity Ledger

Use this file when the main problem is keeping a long manuscript coherent.

## What To Track

Track only state that can later contradict:

- time and chronology
- injuries and recovery
- possession and object location
- power limits and costs
- relationship status
- faction allegiance
- secrets and false beliefs
- promises, debts, and pending consequences

## Ledger Structure

Use flat entries that are easy to update:

```text
Category:
Current fact:
Last confirmed in:
What changed:
What must remain true next:
Risk of contradiction:
```

## High-Risk Areas

Pay special attention to:

- travel times
- training gains
- legal or political fallout
- public versus private knowledge
- romance status drift
- wounds vanishing between scenes
- world rules that bend only when convenient

## Continuity Pass Order

1. Timeline
2. Character state
3. Knowledge state
4. Object and place state
5. Promises and payoffs

Do not do prose polish before this layer is stable if the problem is structural contradiction.

## Compression Rule

If the ledger becomes unreadable, split it into:

- timeline ledger
- cast state ledger
- world rule ledger
- payoff ledger

The goal is reliable checking, not one giant document.
