# Genre Selection Decision Sheet

Use this file when you want a one-page working sheet for locking the genre package quickly without rereading the full rules file.

## Fast Rule

Lock the package that best explains next-chapter return pressure.

If the user directly names a supported genre or a label that maps cleanly to one supported package, keep that package.

If that direct label creates structural friction, do not reclassify. Keep the package and record a warning.

## Decision Sheet

```text

Best match by middle expansion:

Package selection reason:
Alias resolution note:
Structural warning:
```

## Matching Guide

- intimacy, confession, jealousy, attachment pressure -> `romance / romance fantasy`
- advancement, mastery, rank climb, challenge clear -> `progression / action fantasy`
- truth, clue meaning, culprit, reveal timing -> `mystery / thriller`
- factions, decrees, alliances, leverage, public consequence -> `political / war / faction drama`
- scarcity, contamination, pursuit, shelter loss, survival outcome -> `survival / apocalypse / horror`

## Lock Rule

- if the three match lines agree, lock that package
- if they disagree, let return engine win
- if return engine is unclear, let volume-end promise break the tie
- if the raw label is ambiguous, keep it visible in `Alias resolution note`
- if the raw label is a direct user label, keep the mapped package even when another package fits more cleanly

## Warning Rule

Add `Structural warning:` only when all three are true:

- the user directly named the genre or package
- the mapped package is still being kept
- return engine or expansion logic pulls clearly toward another package

Good warning:

- `Structural warning: The user asked for political drama, but the current engine behaves more like romance fantasy unless faction consequence is strengthened.`

Bad warning:

- `Structural warning: The setting is fantasy.`
- `Structural warning: There are monsters.`
