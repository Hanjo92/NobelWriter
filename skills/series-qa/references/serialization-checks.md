# Serialization Checks

Use this file when the work is released or planned as episode-by-episode fiction.

## Audit These Installment Risks

- hooks that repeat the same promise shape
- cliffhangers that resolve with little cost
- recap load crowding out fresh movement
- episodes that tease a turn but mainly stall
- reward spacing that feels too delayed for weekly or daily reading

## Distinguish Retention From Story Health

An episode can end loudly and still weaken the story. Check both:

- does the hook make the next episode feel necessary
- does the next episode cash that necessity with actual change

If the answer to the second question is repeatedly no, the series is burning trust for shallow retention.

## Fast Serial Check

For a quick checkpoint, ask:

1. What changed in this episode besides information volume
2. What consequence from the prior hook is still active
3. What reader promise does the ending make
4. Does the next episode keep or cheapen that promise

## Repair Direction Rule

Report serial-health failures as needing consequence carryover, reduced recap burden, and varied end-hook shape rather than louder episodes.
